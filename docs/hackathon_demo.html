👁️ Observer Agent

Polls Gmail (unread emails), Google Calendar (next 3 days), and a sandboxed filesystem every 60 seconds via MCP tools. Deduplicates events across restarts using Redis SETNX. Detects urgency keywords for priority boosting. Can be triggered manually via dashboard.

🧠 Planner Agent

Consumes events from Redis queue. Calls OpenRouter LLM with structured JSON schema, urgency context, and ChromaDB preferences. Produces action plans with confidence 0–100. Applies decision scoring: adjusted = base × urgency_mult × history_mult

⚡ Executor Agent

Routes by confidence: >90% auto-executes + notifies, 70–89% → dashboard + voice call, <70% silently discards. Records every outcome in ChromaDB so future confidence scoring improves over time.

3 · Voice Approval — The "Wow" Feature

This is the feature that makes judges stop and pay attention. Instead of opening an app and clicking a button, you pick up your phone and talk. The AI understands what you say — approve, reject, or modify with a spoken instruction — and acts in real time.

📧 Email arrives ("reply to professor's urgent deadline email") ↓ Planner scores it: confidence = 78% → requires voice approval ↓ Executor stores plan in Redis voice:plan:{id} with 5-min TTL ↓ Twilio places outbound call to your phone url=/api/twilio/voice/{id} ↓ You answer: "Hey! Requested action: send email. Subject: Assignment Deadline. Say yes to approve, no to reject, or modify followed by instructions." ↓ You say: "reply that I will submit it by 6pm today, please give me some more time" ↓ Twilio POSTs SpeechResult to POST /api/twilio/speech/{id} ↓ LLM classifies intent: MODIFY — instruction: "I will submit it by 6pm today, please give me some more time" ↓ plan["action_args"]["body"] = voice instruction injected into email body plan["approved_override"] = True → re-queued to approvals:pending ↓ Executor auto-executes: send_email(to, subject, body_with_voice_instruction) ↓ You hear: "Got it! I will send email with your changes. Goodbye!" ↓ 📊 Dashboard updates live via WebSocket broadcast

Intent Classification — 4 states

What you sayIntentAction taken
"Yes", "Sure", "Go ahead", "Do it"APPROVEExecutes immediately, email sent
"No", "Don't", "Cancel", "Reject"REJECTRemoved from dashboard, logged as rejected
"Modify …", "Reply that …", "But make it …"MODIFYVoice instruction injected into email body, then executed
Mumble / background noise / silenceUNCLEARRe-prompts you to speak clearly

Fallback safety: If LLM fails, word-boundary regex fallback takes over. "No" does not match "know" or "notify" — uses \b regex to prevent false triggers. speechTimeout="10" gives you 10 full seconds to speak. Say "done" or "stop" to end your instruction.

4 · Smart Meeting Intelligence

PersonalOS Agent now reads your calendar and cross-references it against incoming meeting request emails. It automatically drafts a polished reschedule or confirmation email based on what your calendar actually shows — no manual checking required.

📧 Email arrives: "Can we meet tomorrow at 3pm?" ↓ Observer detects meeting keywords → pushes event to queue ↓ Planner._build_prompts() calls read_calendar(days_ahead=3) via MCP ↓ MeetingResolver parses busy slots + finds free 1-hour windows Conflict check: Is tomorrow 3–4pm busy? ↓ YES → inject calendar context + free slots into LLM prompt ↓ LLM drafts polite reschedule email with specific alternate times ↓ Executor routes to voice approval → you approve by speaking → email sent

# 2. Set up environment (minimum config)
cp .env.example .env
# Edit .env — set OPENROUTER_API_KEY at minimum
# For voice: add TWILIO_ACCOUNT_SID, TWILIO_AUTH_TOKEN,
#            TWILIO_FROM_NUMBER, TWILIO_TO_NUMBER, TWILIO_WEBHOOK_BASE_URL

# 3. Google OAuth (one-time)
python scripts/setup_google_auth.py

# 4. Start Redis
redis-server

# 5. Run pre-flight check
python scripts/check.py

# 6. Launch everything
python main.py

# Open http://localhost:8080

For voice approval, also run ngrok http 8000 and set the printed URL as TWILIO_WEBHOOK_BASE_URL in .env, then restart.
